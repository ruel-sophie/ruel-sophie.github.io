Stage EDU-TECH · Jan–Fév 2026
Supervision réseau avec Zabbix

Déploiement complet d'une infrastructure de supervision réseau via SNMP sur Zabbix 7.4.6. Supervision du switch D-Link DGS-1210-24 et du pare-feu USG FLEX : 207 métriques, 142 triggers, tableaux de bord personnalisés et alertes temps réel.

① Installer et configurer Zabbix 7.4.6

Déploiement du serveur Zabbix 7.4.6 sur Linux. 2 hôtes actifs, 357 modèles disponibles, 296 éléments actifs, 142 déclencheurs configurés.

  • Installer Zabbix server, frontend et agent sur le serveur Linux
  • Configurer la base de données MySQL et le service Apache
  • Accéder à l'interface web et terminer l'assistant de configuration
  • Vérifier : serveur en cours d'exécution sur localhost:10051
  • Résultat : version 7.4.6, 357 modèles chargés, 2 hôtes supervisés
② Créer et configurer l'hôte SW-DLINK-1210-24

Ajout du switch D-Link DGS-1210-24 via SNMP. IP 192.168.10.185 port 161. Modèle Generic by SNMP appliqué.

  • Configuration → Hôtes → Créer un hôte
  • Nom : SW-DLINK-1210-24 · Interface : SNMP · IP 192.168.10.185 · Port 161
  • Sélectionner le modèle Generic by SNMP
  • Sauvegarder — Zabbix commence à collecter les données
③ Sélectionner le bon template SNMP

Navigation dans Templates/Network devices. Le template Generic by SNMP collecte ICMP, SNMP availability, System info, Uptime et interfaces réseau.

  • Cliquer sur Sélectionner dans le champ Modèles
  • Naviguer : Templates → Templates/Network devices
  • Choisir Generic by SNMP pour les métriques de base
④ Explorer les Dernières données collectées

207 éléments organisés en tags. Ports Slot0/1 à Slot0/20 monitorés. Uptime : 211 jours. Métriques : bits reçus/émis, duplex, vitesse par port.

  • Monitoring → Dernières données → Filtrer par hôte SW-DLINK-1210-24
  • 207 éléments visibles : interfaces, system info, ICMP
  • Graphiques disponibles pour chaque port réseau
⑤ Supervision du pare-feu USG FLEX

Ajout du pare-feu USG FLEX via SNMP avec template Zyxel dédié. Supervision CPU, mémoire, sessions actives et interfaces WAN/LAN.

  • Créer l'hôte USG-FLEX avec interface SNMP
  • Appliquer le template Zyxel USG FLEX by SNMP
  • Configurer les triggers : CPU > 80%, mémoire > 90%
  • Créer les alertes mail pour les incidents critiques
⑥ Tableaux de bord personnalisés

Création de dashboards Zabbix avec widgets graphiques, jauges et statuts en temps réel pour une vision globale de l'infrastructure réseau.

  • Monitoring → Tableaux de bord → Créer un tableau de bord
  • Ajouter widgets : graphiques trafic, statut hôtes, uptime, problèmes actifs
  • Configurer le rafraîchissement automatique toutes les 30 secondes
  • Partager le dashboard avec l'équipe technique

Sauvegarde externalisée NAS

Mise en place d'un système de sauvegarde externalisée sur NAS pour garantir la continuité de service et la récupération des données en cas d'incident. Stratégie de Plan de Reprise d'Activité (PRA).

① Architecture de sauvegarde

Solution NAS externalisée pour protéger les données critiques de l'entreprise. Politiques de sauvegarde planifiées avec versioning et rétention.

  • Analyser les besoins de sauvegarde et définir les données critiques
  • Configurer le NAS avec RAID pour la redondance
  • Définir les planifications de sauvegarde incrémentale / complète
  • Tester la procédure de restauration
② Configuration et tests

Paramétrage des règles de rétention, tests de restauration et documentation du Plan de Reprise d'Activité pour l'équipe technique.

  • Configurer les partages réseau et les droits d'accès
  • Planifier les sauvegardes automatiques (nuit / weekend)
  • Tester la restauration d'un fichier et d'un dossier complet
  • Rédiger la documentation PRA pour l'équipe
